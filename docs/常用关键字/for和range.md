# Go 语言 for range 实现原理、版本演进与使用指南

## 1. 核心原理

`for range` 循环是 Go 提供的语法糖，编译器会将其转换为普通的 `for` 循环。转换方式取决于遍历的数据类型（数组、切片、字符串、map、channel）。

## 2. 不同数据结构的遍历实现

### 2.1 数组 / 切片


```go
// 原始代码
for i, v := range slice {
    // ...
}
```

**编译器转换后的伪代码**：

```go
len := len(slice)
for i := 0; i < len; i++ {
    v := slice[i]   // 复制元素值到 v
    // ...
}
```

- `i` 是索引，`v` 是元素的**值拷贝**。
- 循环开始前会先计算长度，因此循环中添加/删除元素不会影响迭代次数。

### 2.2 字符串

```go
for i, ch := range str {
    // ch 是 rune 类型
}
```

- 遍历的是 Unicode 码点（rune），不是字节。
- `i` 是字节索引（可能不连续，因为 UTF-8 编码），`ch` 是 `rune` 类型。
- 底层解码使用 `utf8.DecodeRuneInString`，无效 UTF-8 时 `ch = 0xFFFD`，索引前进 1。

### 2.3 map
![map遍历](https://img.draven.co/2020-01-17-15792766877639-golang-range-map.png)

```go
for k, v := range m {
    // ...
}
```

- 遍历顺序**随机**（Go 1.12+ 有意识增加随机性），不能依赖顺序。
- 在迭代过程中**删除键或新增键**是安全的，但新增的键不保证在当前迭代中出现。
- 编译器转换为调用 `runtime.mapiterinit` 和 `runtime.mapiternext`。
**哈希表遍历的顺序，首先会选出一个绿色的正常桶开始遍历，随后遍历所有黄色的溢出桶，最后依次按照索引顺序遍历哈希表中其他的桶，直到所有的桶都被遍历完成。**
![map遍历顺序](https://img.draven.co/2020-01-17-15792766877646-golang-range-map-and-buckets.png)

### 2.4 channel

```go
for v := range ch {
    // 从 channel 接收值，直到 channel 被关闭
}
```

- 等价于 `for { v, ok := <-ch; if !ok { break } }`。
- 如果 channel 为 `nil`，循环会永久阻塞。
- 必须显式 `close(ch)` 才能终止循环。

## 3. 新老版本设计区别

| 方面 | Go 1.16 及之前 | Go 1.17 及之后 |
|------|----------------|----------------|
| **map 遍历随机性** | 起始桶随机，但顺序相对固定（与插入顺序有关） | 增加伪随机因子，每次遍历顺序几乎完全不同，防止依赖 |
| **数组/切片复制优化** | 正式遍历数组前会复制整个数组到临时变量（若数组大则开销大） | 无变化，但编译器提示更明确；建议用切片或指针避免复制 |
| **channel 遍历** | 无变化 | 无变化 |
| **闭包捕获循环变量陷阱** | 循环变量复用地址，常犯错误 | 无变化，依然是复用；但 Go 1.22 将改变（见注意事项） |

> 注：Go 1.22 计划将 `for` 循环的变量作用域改为每次迭代独立，解决闭包陷阱。但目前仍按复用处理。

## 4. 使用注意事项

### 4.1 循环变量地址问题（经典陷阱）

```go
func main() {
    s := []int{1, 2, 3}
    var ptrs []*int
    for _, v := range s {
        ptrs = append(ptrs, &v)
    }
    for _, p := range ptrs {
        fmt.Print(*p) // 输出 333
    }
}
```

- 原因：`v` 只声明一次 ，每次迭代复用同一个变量，地址不变。
- **应对**：使用 `v := v` 或直接取 `&s[i]`。

### 4.2 遍历 map 时顺序不确定

- 不要依赖 map 遍历顺序，若需固定顺序，先取出 key 排序。

### 4.3 遍历数组会复制整个数组

```go
var arr [1024]int
for i, v := range arr { // 复制整个数组
    // ...
}
```
- **应对**：使用切片 `arr[:]` 或数组指针 `&arr`。

### 4.4 遍历过程中修改切片

- 添加或删除元素**不会影响迭代次数**，因为长度在循环前已确定。
- 修改已遍历索引的元素会影响后续迭代（因为底层数组共享）。

### 4.5 遍历 channel 必须关闭

- 忘记 `close(ch)` 会导致 `deadlock` 或永久阻塞。
- 使用 `for v := range ch` 前确保 channel 会在某个时刻关闭。

### 4.6 遍历 nil channel

- `for v := range nil` 永久阻塞，不会 panic。

### 4.7 遍历字符串时注意索引是字节位置

```go
s := "你好"
for i, ch := range s {
    fmt.Println(i, ch) // i 为 0 和 3
}
```

### 4.8 性能：避免在循环内做昂贵操作

- `range` 循环每次迭代会复制元素值，若元素很大（如大结构体），使用索引访问或指针切片。

## 5. 使用场景

| 数据类型 | 常见场景 | 注意事项 |
|----------|----------|----------|
| **数组/切片** | 遍历所有元素进行读取或修改 | 修改元素需用 `s[i]`；避免值拷贝大结构体 |
| **字符串** | 处理 Unicode 字符（如中英文混合） | 注意索引是字节位置，不是字符位置 |
| **map** | 遍历键值对，尤其是需要读取全部数据时 | 顺序随机；遍历中安全删除/新增 |
| **channel** | 持续消费 channel 中的数据直到关闭 | 必须确保 channel 关闭，否则死锁 |

### 5.1 典型代码示例

**读取 channel 直到关闭**：
```go
func consumer(ch <-chan int) {
    for v := range ch {
        fmt.Println(v)
    }
    fmt.Println("channel closed")
}
```

**遍历 map 并删除不符合条件的项**：
```go
for k, v := range m {
    if v < 0 {
        delete(m, k)
    }
}
```

**遍历字符串获取字符**：
```go
for i, ch := range "Hello, 世界" {
    fmt.Printf("%d: %c\n", i, ch)
}
```

## 6. 总结

`for range` 是 Go 中遍历复合类型的高效语法，编译器针对不同数据结构生成不同的底层循环逻辑。使用时需注意循环变量复用、值拷贝开销、map 随机顺序、channel 关闭等陷阱。新版本（Go 1.17+）主要增强了 map 遍历的随机性，Go 1.22 将修复循环变量闭包陷阱。

> 注：本文档基于 Go 1.18~1.22 源码分析，具体实现细节可能随版本变化，建议查看目标版本的 `src/cmd/compile/internal/range.go` 及相关运行时函数。
